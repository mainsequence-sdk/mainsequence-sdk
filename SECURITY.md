# Security Policy

## Reporting a Vulnerability

If you believe you have found a security vulnerability in this project, do not report it through a public GitHub issue.

Instead, report it privately to:

- `dev@main-sequence.io`

Please include:

- a clear description of the issue
- affected package version(s)
- reproduction steps or a proof of concept
- impact assessment if known
- any suggested mitigation or fix

We will review reports as quickly as practical and follow up with next steps once the issue has been triaged.

## Supported Versions

Security fixes are prioritized for:

- the latest published release
- the current main development branch

If you are running an older version, upgrade to the latest supported release before requesting a fix whenever possible.

## Disclosure Guidance

- Please avoid disclosing sensitive details publicly before a fix or mitigation is available.
- If a report turns out not to be security-sensitive, it may be redirected to the normal issue tracker.
